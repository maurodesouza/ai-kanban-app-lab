## Project Overview

This project is a simple Kanban application built as a controlled experiment to understand how to guide AI in a real development workflow.

The main goal is not the Kanban app itself, but the process of building it using AI with minimal manual intervention. The focus is to evaluate how the AI interprets instructions, follows rules, and handles architectural decisions over time.

### Objectives

* Understand how AI behaves in a structured development environment
* Evaluate consistency when following predefined rules and patterns
* Identify common failure points (architecture, typing, composition, etc.)
* Improve prompt design and development workflow with AI

### Scope

* A simple Kanban board (columns, tasks, basic interactions)
* No unnecessary features beyond what is explicitly requested
* Emphasis on structure, not feature complexity

### Constraints

* Minimize manual code changes
* Prefer guiding the AI over fixing its output
* All decisions must be driven by clear instructions and rules
