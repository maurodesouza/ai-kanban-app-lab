# AI Kanban App Lab

This repository is a set of experiments where I try to build a Kanban app using AI.

I am using [Windsurf IDE](https://windsurf.com) (Pro plan) and following the [Spec Kit](https://github.com/github/spec-kit) process from GitHub.

The goal is to understand how to work with AI in a real development workflow, minimizing manual intervention and evaluating the raw output generated by the AI.

## Process & Result


https://github.com/user-attachments/assets/d84b0c17-d498-4b75-a1a0-cf104b3b1ddd


This first attempt was not good.

The final app was messy and poorly structured. The AI created many things that I did not ask for and ignored most of my rules.

It did not follow almost any of the architectural decisions I defined, including:

* event organization
* styling system (tokens)
* component structure
* component composition

The only rule it consistently followed was writing documentation in English.

I followed the Spec Kit process, but I misunderstood how to use it. I tried to describe the entire application in a single specification, which created too much ambiguity.

During the process, I made two different mistakes:

At the beginning, I tried to correct the AI by editing the code manually when something was wrong. This created inconsistency, because the AI lost track of the real state of the project.

Later, I stopped interfering and let the AI generate things more freely, without strong supervision. This also did not work well, and the project became even more uncontrolled.

In the end, I had to manually fix 2 runtime issues and 1 styling issue to make the application run. Other issues were intentionally ignored, as the goal was to keep manual fixes to a minimum and evaluate the raw AI output.

## Observations

* I should create much smaller specifications before asking the AI to work
* I should start with a base structure, not let the AI design everything alone
* The AI does not follow most architectural rules consistently
* Maybe the SWE-1.5 model is not very good for this type of task
* Sometimes the AI asks to run a command, but it gets stuck in an infinite loop in the terminal
* Editing code manually during the process breaks consistency with the AI
* Letting the AI work without supervision also leads to bad results
